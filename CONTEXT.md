# PAE Contable — Frontend Context

Next.js 14 frontend for the Colombian accounting automation platform. Consumes the FastAPI backend via domain-specific HTTP clients.

## Language

**ApiClient**:
The seam between the frontend and the backend — a domain-scoped HTTP adapter that encapsulates request/response logic for a single business area (e.g., tax, ingest, reports).
_Avoid_: service, gateway, helper

**Módulo**:
A major functional area of the UI, identified by a number (1–10) and a canonical route. Examples: Módulo 1 Dashboard, Módulo 2 Upload, Módulo 4 Reports, Módulo 6 Tax.
_Avoid_: page, section, feature

**Empresa activa**:
The company currently selected by the user, identified by NIT. All data fetching is scoped to this NIT.
_Avoid_: active company, current org

**Vía A**:
Upload flow for source documents (PDF/XML/Excel) that triggers the full accounting pipeline.
_Avoid_: upload mode A, source flow

**Vía B**:
Upload flow for pre-existing financial statements (balance general, estado de resultados, libro auxiliar) that bypasses the accounting pipeline.
_Avoid_: upload mode B, statement flow

**Proceso**:
A long-running backend job (ingest or accounting pipeline) that the frontend polls for status. Identified by `process_id`.
_Avoid_: job, task, run

**Ingesta**:
The backend pipeline phase that extracts data from uploaded documents. The frontend tracks it via `ingest_id`.
_Avoid_: upload processing

**Borrador**:
A tax declaration draft (e.g., F300, F350) generated by the backend and editable in the frontend.
_Avoid_: draft, form

**Traza**:
The audit trace returned by the backend for a completed proceso, showing each agent step, findings, and blockers.
_Avoid_: trace, log, audit trail

**Estado financiero**:
A financial statement rendered in the UI: balance general, estado de resultados, libro auxiliar, flujo de caja, etc.
_Avoid_: financial report, statement

## Relationships

- An **Empresa activa** (NIT) scopes all **ApiClient** requests
- **Vía A** and **Vía B** are two upload pathways within the same **Módulo 2** (Upload)
- A **Proceso** has exactly one **Traza**, exposed after completion
- **Módulo 6** (Tax) consumes **Borradores** produced by the backend
- **Módulo 4** (Reports) renders **Estados financieros** fetched via **ReportApiClient**

## Example dialogue

> **Dev:** "When the user uploads a factura via Vía A, which ApiClient handles the polling?"
> **Domain expert:** "The `IngestApiClient` starts the upload and returns an `ingest_id`. The `ProcessApiClient` then polls the proceso status until it completes or fails."

## Flagged ambiguities

- "Upload" was used to mean both the file drop action and the backend ingest pipeline. Resolved: "upload" is the UI action; **Ingesta** is the backend pipeline.
- "Company" was used interchangeably for the Supabase auth entity and the accounting entity. Resolved: **Empresa activa** refers to the accounting company selected in the UI.
