## Task 3: Via B — surface timeout error to slot state

**Files:**
- Modify: `src/hooks/useUpload.ts` — `useViaBUpload`, the block that calls `waitForDerivedStatements`

- [ ] **Step 1: Find the call site**

Open `src/hooks/useUpload.ts`. Search for `waitForDerivedStatements`. It is called in the Via B upload flow after all slots are individually uploaded/polled.

- [ ] **Step 2: Wrap in try-catch**

The current code likely looks like:

```ts
await waitForDerivedStatements(activeNit ?? '');
setSlots(...done...);
```

Wrap it:

```ts
try {
    await waitForDerivedStatements(activeNit ?? '');
    // existing done-setting setSlots call stays here
} catch (err: unknown) {
    const message = extractErrorMessage(err);
    setSlots((prev) =>
        prev.map((s) =>
            s.status === 'extracting' || s.status === 'uploading'
                ? {
                      ...s,
                      status: 'error',
                      error: message,
                      error_category: 'timeout',
                      remediation: 'Los estados financieros derivados tardaron demasiado. Vuelve a intentarlo.',
                  }
                : s
        )
    );
}
```

`extractErrorMessage` is already defined in this file (line ~72).
