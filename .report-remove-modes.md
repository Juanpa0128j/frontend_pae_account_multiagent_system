## Type location

No `ParserMode` union type exists. `parser_mode` is typed as bare `string` everywhere (src/types/index.ts:251, src/hooks/useUpload.ts:938, component props). The only place `'premium'`/`'gpt4o'` existed as concrete values was the `MODES` array literal in the picker component.

## Sites changed

1. `src/components/upload/BrutalistParsingSelector.tsx` — removed the `premium` ("Alta calidad") and `gpt4o` ("Máxima precisión") entries from the `MODES` array. Remaining 4 canonical entries: `fast`, `standard`, `agentic`, `agentic_plus`. No default value hardcoded to `premium` (default already computed via `MODES[0]` = `fast`).
2. `src/test/components/BrutalistParsingSelector.test.tsx`:
    - "renders all six modes" test → now expects 4 options, dropped assertions for "Alta calidad"/"Máxima precisión" labels.
    - "calls onChange when option clicked" test → switched fixture from clicking "Alta calidad" (premium) to clicking "Estándar" (standard); assertion now `onChange` called with `'standard'`.
3. `src/test/hooks/useUploadReorderQueue.test.tsx` — fixture in "reorderQueue preserves FileUploadState metadata" test used `parser_mode: 'premium'`; replaced with `parser_mode: 'agentic'` (test only checks metadata passthrough, not mode semantics).
4. `src/test/upload/useUpload.test.tsx` — grep flagged, but on inspection its `parser_mode` fixtures already used `'standard'`/`'fast'` — no changes needed.
